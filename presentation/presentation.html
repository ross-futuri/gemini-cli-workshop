Core Concepts: What's Under the Hood?

Let's demystify the key ideas that make this all work.

The Transformer Architecture: Engine & Focus

The **Transformer** is the engine of modern AI. Unlike older models that read word-by-word, it processes entire sentences at once. The breakthrough that makes this possible is the **Attention Mechanism**.

Analogy: The Open-Book Exam. You don't re-read a whole book for one question. You scan for keywords, "paying attention" to relevant parts. The Attention mechanism lets the AI weigh the importance of different words, giving it focus.

AI's Quirks: Creativity & Amnesia

An AI's behavior is shaped by two key concepts: **Non-Determinism** (it makes creative, probabilistic guesses, not rigid calculations) and its **Context Window** (its short-term memory).

Analogy: A Creative Intern with Amnesia. The AI can come up with brilliant, unexpected solutions (creativity), but it only remembers what's on the "whiteboard" of the current conversation. If information scrolls off, it's forgotten (amnesia).

The Unseen Costs of AI

  • A single AI query can use 10-100x more electricity than a Google search.
  • Training one large model can equal the lifetime emissions of five cars.

This isn't to say AI is bad, but we must be mindful. AI is also a tool being used to design more energy-efficient systems.

Installation & Setup

Let's get our hands dirty and prepare our environment.

Setup Part 1: The Foundation (WSL & Node)

First, we need to install the Windows Subsystem for Linux and Node.js.


# 1. Install WSL from PowerShell or CMD (as Admin)
# This command installs the WSL subsystem and a default Ubuntu distribution.
wsl --install

# 2. Launch WSL and install NVM (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 3. Reload the shell (exit and re-launch) and install Node.js
nvm install --lts

Setup Part 2: The Project (Git & Gemini)

Now, let's install Git, the Gemini CLI, and get the workshop files.


# 1. Install Git in your WSL terminal
sudo apt update && sudo apt install git -y

# 2. Install the Gemini CLI globally
npm install -g @google/gemini-cli

# 3. Set your API Key
# Get your key from Google AI Studio and set it as an environment variable
export GEMINI_API_KEY="YOUR_API_KEY_HERE"

# 4. Clone the workshop repository
git clone https://github.com/ross-futuri/gemini-cli-workshop.git
cd gemini-cli-workshop

# 5. Pre-flight Check!
node -v && gemini --version && ls

The Prompt-Driven Workflow

A successful project follows an iterative, collaborative process with the AI.

  • 1. Review the Plan: Start with a clear goal. We'll examine the initial prompt that outlines our web application.
  • 2. Refine with AI: Collaborate with the AI to improve the plan. A good conversation can clarify requirements and prevent mistakes.
  • 3. Supervise Implementation: Ask the AI to build the application, requesting it to mark its progress so you can follow along.
  • 4. Review and Test: Once the code is written, it's your turn. Run the app, test its functionality, and verify the results.

This human-in-the-loop process ensures you remain in control while leveraging the AI's speed.

Working with AI: Rules of Engagement

  • Use Version Control Relentlessly. `git commit` is your best safety net.

    War Story: To fix minor linting issues, an AI wrote a script that destroyed the codebase. I recovered with Git but lost an hour of work.

  • Plan in Markdown. A detailed prompt or plan is your source of truth to keep the AI on track.
  • Maintain a Consistent Environment. WSL can have permission issues; run tests and static analysis in it to match the AI's environment.
  • You Are the Developer. The AI is a partner, not a magician. You must guide and verify its work.

    War Story: An AI stopped using its tools but insisted everything was working. I had to restart it to get it to see its own failure.
