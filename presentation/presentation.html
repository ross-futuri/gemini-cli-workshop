Core Concepts: How It Works

Let's demystify the key ideas that make this all work.

The Transformer & Attention

The **Transformer** is the engine of modern AI. The breakthrough that makes it possible is the **Attention Mechanism**.

Analogy: The Open-Book Exam. You don't re-read a whole book for one question. You scan for keywords, "paying attention" to relevant parts. Attention lets the AI weigh the importance of different words, giving it focus.

Further Reading: The Illustrated Transformer

Creativity & Context

An AI's behavior is shaped by two key concepts: **Non-Determinism** (it makes creative, probabilistic guesses) and its **Context Window** (its short-term memory).

The Context Window is the model's entire short-term memory. It's a set number of tokens the model can consider at one time. Information outside this window is forgotten, which is the most common source of errors.

Further Reading: What is a context window? (IBM)

The Unseen Costs of AI

  • A single AI query can use 10-100x more electricity than a Google search.
  • Training one large model can equal the lifetime emissions of five cars.

This isn't to say AI is bad, but we must be mindful. AI is also a tool being used to design more energy-efficient systems.

Installation & Setup

Let's get our hands dirty and prepare our environment.

Setup: The Foundation (WSL & Node)

We need a consistent environment. Using the Windows Subsystem for Linux is key.


# 1. Install WSL and a specific Ubuntu distribution
wsl --install -d Ubuntu --name Gemini-cli

# 2. Launch our dedicated WSL environment
wsl -d Gemini-cli

# 3. Install NVM (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 4. Reload shell (exit & relaunch) and install Node.js
nvm install --lts

Setup: The Project (Gemini CLI)

Now, let's install the CLI and set our API key.


# 1. Install the Gemini CLI globally
npm install -g @google/gemini-cli

# 2. Set your API Key from Google AI Studio
export GEMINI_API_KEY="YOUR_API_KEY_HERE"

# 3. Pre-flight Check!
node -v && gemini --version

Setup: Get the Workshop Files

Now let's install Git and clone the workshop repository.


# 1. Install Git in your WSL terminal
sudo apt update && sudo apt install git -y

# 2. Clone the workshop repository
git clone https://github.com/ross-futuri/gemini-cli-workshop.git

# 3. Navigate into the project directory
cd gemini-cli-workshop

AI Workflows: From Assistant to Agent

  • Copy/Paste (UI-based): Basic prompting in a web UI and pasting the code over. Simple and contained.
  • Integrated (CLI / MCP): The AI has access to your file system. More efficient, but requires careful oversight. (Today's Focus)
  • Agentic (Unpredictable): You give the AI a high-level goal and let it run. The results are often not worth the effort.

The most reliable pattern is **Plan, Implement, Review.** We plan the work, the AI implements, and we verify.

Pitfalls: Common Failure Modes

  • Context Amnesia: It's like working with a fast but forgetful coder. It has no long-term memory.
  • The "Hidden Prompt": Hidden system instructions can override yours, requiring you to be persistent.

War Story 1: "To fix minor linting issues, an AI wrote a script that destroyed the codebase. It took a simple task and found the most destructive solution."

War Story 2: "An AI stopped using its tools but insisted everything was working. It can't self-diagnose, so I had to restart the session."

Best Practices: Your Safety Net

  • Use Version Control Relentlessly. `git commit` is your only real safety net. The AI *can*, however, be useful for searching git history.
  • Plan in Markdown. An external plan is your source of truth when the AI's context fails.
  • Keep Your Workspace Clean. The AI can get confused by old or duplicate files. Use `git status` to track changes.
  • Remind It Constantly. You have to actively manage the AI's focus to keep it from going off-task.
  • Use a Consistent Environment. Run tests in WSL to match the AI's environment and avoid permission issues.

Advanced Techniques

  • AI-Powered Code Investigation: Use the AI to analyze `git log` or `git blame` output to understand code history and intent.
  • Automated Static Analysis: Feed linter or type-checker output to the AI and instruct it to fix the identified issues.
  • Complex Task Chaining: Use a single prompt to orchestrate multi-step tasks, like reading a spec, generating a client, and writing a test.
  • Infrastructure as Code Generation: Ask the AI to create `Dockerfile` or `docker-compose.yml` files for your application.
